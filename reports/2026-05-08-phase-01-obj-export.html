03 / Algorithm

Algorithm

The OBJ export uses FreeCAD's Mesh module to triangulate each document object. Objects are iterated in document order, each written as an OBJ group with a name derived from its label and floor level prefix (e.g. F0_Wall_ext_left_2D). A material inference function (_infer_material_key) classifies objects by keywords in their label to assign materials from a palette.

Each per-floor OBJ gets its own MTL file; the combined model uses a shared tubehouse.mtl. Vertex indices in the OBJ use 1-based counting with an offset tracker across objects.

04 / Tools

Tools & Methods

  • FreeCAD Mesh module — triangulates Part shapes into indexed face sets for OBJ export
  • Wavefront OBJ format — chosen over FBX (simpler) and IFC (heavier, phase-2): preserves group names for Blender material assignment
  • MTL material library — minimal principled material definitions with Kd, Ka, Ks, Ns, d, illum
  • Python unittest — tests for pure helpers (group naming, path construction, MTL generation) without FreeCAD dependency
  • Convention-based material inference — label-keyword matching instead of hardcoding, so new object types auto-classify

06 / Limitations

Limitations / Second-Best Alternative

  • No runtime verification yet. OBJ export depends on FreeCAD's Mesh module which cannot run outside FreeCAD. Tests cover pure helpers only; actual OBJ file generation requires a FreeCAD runtime.
  • OBJ group names depend on FreeCAD label conventions. If object labels change in future phases, material inference may misclassify. The sidecar config (spec/blender_materials.json in PHASE-02) addresses this.
  • STL fallback exists but drops per-object naming entirely. OBJ was chosen specifically because it preserves group structure for Blender.
  • FBX was considered as an alternative but rejected due to version-compatibility issues between FreeCAD's exporter and Blender's importer.
07 / Errors & Warnings

Errors / Warnings / Flags

  • Windows path separator in tests. Initial test used /tmp/out/obj/floorplan_F0.obj as expected path, but os.path.join on Windows produced backslash separators. Fixed by normalizing with .replace("\\", "/") in assertions.
  • Unclosed file handles in MTL tests. open().read() pattern triggered ResourceWarning. Fixed by using with open() as fh context managers.
  • No errors in py_compile checks. All five source and test files compile cleanly.
