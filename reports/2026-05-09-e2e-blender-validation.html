06 / Limitations / Second-Best Alternative

Limitations / Second-Best Alternative

  • Facade SVG missing: FreeCAD's SVG export requires FreeCADGui which is unavailable in headless mode. Facade DXF and FCStd exported correctly; SVG cannot be produced without a GUI session.
  • PDF export blocked: The cairosvg module is not available in the FreeCAD Python environment, preventing SVG-to-PDF conversion for all floors.
  • F4 DXF encoding issue: Floor 4 DXF export failed due to a Unicode em-dash (\u2014) that can't be encoded in latin-1. Floor plan labels need ASCII-only characters for DXF.
  • Combined OBJ file size is small (3.2 KB): The combined OBJ contains geometry groups but may have low vertex counts. Future work could increase mesh resolution in FreeCAD before export.
  • Alternative approach considered: We could have used FBX or IFC for interchange, but OBJ was chosen for simplicity and Blender compatibility. STL was generated as a fallback geometry format.

08 / Open Questions / Next Phase Seeds

Open Questions / Next Phase Seeds

  • Visual quality iteration: The first Cycles render is functional but basic. Camera angle, material tweaks, lighting adjustments, and higher sample counts would improve visual fidelity.
  • SCRIPT_DIR fallback path: The hardcoded fallback in generate_floorplan.py still references freecad-floorplan. This should be removed or updated to use the actual project directory.
  • PDF pipeline: Install cairosvg in FreeCAD's Python environment to enable SVG-to-PDF conversion.
  • Facade SVG: Requires a GUI FreeCAD session. Could be addressed via freecad-mcp with a running FreeCAD GUI, or by writing a pure-Python SVG generator.
  • F4 DXF encoding: Replace em-dashes and other non-latin-1 characters in spec labels with ASCII equivalents.
  • Mesh resolution: Current OBJ exports have relatively low vertex counts. Future iterations could use FreeCAD's Mesh module with higher resolution settings before OBJ export.
